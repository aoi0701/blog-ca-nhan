Trong lập trình mạng, trước khi dữ liệu có thể được truyền đi, **hai thiết bị cần biết chính xác địa chỉ của nhau**.  
Việc xác định **địa chỉ kết nối mạng** là bước đầu tiên và quan trọng nhất trong mọi ứng dụng mạng như:

- Client – Server
- Chat application
- Truyền file
- Web server
- IoT

Bài viết này giúp bạn hiểu **quản lý địa chỉ kết nối mạng là gì**, các thành phần cấu thành địa chỉ mạng và cách Java hỗ trợ quản lý chúng.

## 1. Địa chỉ kết nối mạng là gì?

![DNS](../../images/08.jpg)

<p align="center">
  <em>Sơ đồ kiến trúc mạng doanh nghiệp, bao gồm Internet, tường lửa, hệ thống máy chủ, thiết bị lưu trữ và mạng nội bộ (LAN).</em>
</p>

**Địa chỉ kết nối mạng** là thông tin giúp một thiết bị:
- Xác định **thiết bị đích**
- Xác định **dịch vụ cụ thể** trên thiết bị đó

Một kết nối mạng đầy đủ thường có dạng:

```

IP Address : Port

```

Ví dụ:
```

192.168.1.10 : 8080

```

---

## 2. Địa chỉ IP (Internet Protocol Address)

### 2.1 IP là gì?

**Địa chỉ IP** là định danh duy nhất của một thiết bị trong mạng.

> Có thể hiểu IP giống như **địa chỉ nhà**, giúp xác định vị trí thiết bị trên mạng.

---

### 2.2 IPv4 và IPv6

#### IPv4
- Dạng: 4 số, mỗi số từ 0–255
- Ví dụ: `192.168.1.1`
- Phổ biến hiện nay

#### IPv6
- Dạng: hệ 16
- Ví dụ: `2001:db8::1`
- Sinh ra để thay thế IPv4 do thiếu địa chỉ

---

### 2.3 IP Public và IP Private

| Loại IP | Mục đích |
|------|---------|
| Public IP | Truy cập Internet |
| Private IP | Mạng nội bộ (LAN) |

Một số dải IP private phổ biến:
- `192.168.x.x`
- `10.x.x.x`
- `172.16.x.x – 172.31.x.x`

---

## 3. Port – Cổng kết nối

### 3.1 Port là gì?

**Port** là số hiệu dùng để xác định **ứng dụng hoặc dịch vụ cụ thể** trên một thiết bị.

> Nếu IP là địa chỉ nhà thì **Port là số phòng**.

---

### 3.2 Phân loại Port

| Khoảng | Ý nghĩa |
|-----|--------|
| 0 – 1023 | Well-known ports |
| 1024 – 49151 | Registered ports |
| 49152 – 65535 | Dynamic ports |

Ví dụ:
- HTTP: 80
- HTTPS: 443
- FTP: 21

---

## 4. DNS – Phân giải tên miền

![DNS](../../images/04.jpg)

### 4.1 DNS là gì?

**DNS (Domain Name System)** giúp chuyển:
```

Tên miền → Địa chỉ IP

```

Ví dụ:
```

[www.google.com](http://www.google.com) → 142.250.72.36

````

DNS giúp con người **không cần nhớ IP số**, chỉ cần nhớ tên miền.

---

## 5. Quản lý địa chỉ mạng trong Java

Java cung cấp lớp **InetAddress** để làm việc với địa chỉ IP.

### 5.1 Lớp InetAddress

`InetAddress` cho phép:
- Lấy IP từ hostname
- Lấy hostname từ IP
- Kiểm tra địa chỉ mạng

---

### 5.2 Ví dụ lấy IP từ tên miền

```java
import java.net.InetAddress;

public class GetIP {
    public static void main(String[] args) throws Exception {
        InetAddress address = InetAddress.getByName("google.com");
        System.out.println("IP: " + address.getHostAddress());
    }
}
````

---

### 5.3 Lấy thông tin máy local

```java
InetAddress local = InetAddress.getLocalHost();
System.out.println("Host name: " + local.getHostName());
System.out.println("IP: " + local.getHostAddress());
```

---

## 6. Địa chỉ kết nối trong Client – Server

Trong lập trình mạng Java:

* **Server**

  * Lắng nghe tại một **port**
  * Có IP (hoặc localhost)

* **Client**

  * Kết nối đến **IP + Port** của server

Ví dụ:

```java
Socket socket = new Socket("localhost", 1234);
```

---

## 7. Các lỗi thường gặp khi quản lý địa chỉ mạng

### 7.1 Sai IP hoặc Port

* Kết nối không được
* Bị `ConnectException`

---

### 7.2 Port đã được sử dụng

* Lỗi: `BindException`
* Do port đang được chương trình khác dùng

---

### 7.3 DNS không phân giải được

* Sai tên miền
* Không có Internet

---

## 8. Ứng dụng thực tế

Quản lý địa chỉ kết nối mạng xuất hiện trong:

* TCP / UDP Socket
* Web Server
* Chat application
* IoT
* Game online
* Hệ thống phân tán

---

## 9. Tổng kết

* Địa chỉ kết nối mạng gồm **IP + Port**
* IP xác định thiết bị, Port xác định dịch vụ
* DNS giúp chuyển tên miền thành IP
* Java hỗ trợ quản lý địa chỉ qua `InetAddress`
* Đây là kiến thức nền tảng để học TCP / UDP
